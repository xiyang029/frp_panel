# 项目文件架构

本文说明当前仓库的主要文件与目录职责。需要定位代码时，先按本文件缩小范围，再读取具体实现文件。

## 根目录文件

- `AGENTS.md`：项目级 Agent 协作规范、构建测试命令和结构索引入口。
- `README.md`：英文项目介绍、功能说明、使用示例和贡献说明。
- `README_zh.md`：中文项目介绍、功能说明、使用示例和贡献说明。
- `Release.md`：发布相关说明。
- `LICENSE`：项目许可证。
- `go.mod`、`go.sum`：Go 模块定义与依赖锁定。
- `Makefile`：本地构建、测试、格式化、Web 面板构建和清理入口。
- `Makefile.cross-compiles`：跨平台编译相关目标。
- `package.sh`：发布打包脚本。
- `.golangci.yml`：Go lint 规则配置。
- `.gitignore`：Git 忽略规则。
- `.goreleaser.yml`：GoReleaser 发布配置。
- `CLAUDE.md`：外部工具提示文件，当前内容极少，不承载项目结构说明。

## 核心 Go 代码

- `cmd/frps/`：`frps` 服务端命令入口、根命令和配置校验。
- `cmd/frpc/`：`frpc` 客户端命令入口。
- `cmd/frpc/sub/`：客户端子命令，包括代理管理、NAT hole、admin 和配置校验。
- `server/`：服务端运行时核心，负责接收客户端控制连接、注册代理、监听端口、管理访问者和对外 HTTP API。
- `server/proxy/`：服务端各类代理实现，包括 TCP、UDP、HTTP、HTTPS、STCP、SUDP、XTCP、TCPMUX。
- `server/group/`：服务端代理分组、监听器和负载均衡相关逻辑。
- `server/http/`：服务端管理 API 控制器与响应模型。
- `server/registry/`：服务端代理和访问者注册表。
- `server/ports/`：端口分配与占用管理。
- `server/metrics/`：服务端指标采集。
- `server/controller/`：服务端资源控制与协调逻辑。
- `server/visitor/`：服务端访问者相关实现。
- `client/`：客户端运行时核心，负责连接服务端、维护控制会话、加载配置、管理代理和访问者。
- `client/proxy/`：客户端各类代理实现和代理管理器。
- `client/visitor/`：客户端 STCP、SUDP、XTCP 访问者实现。
- `client/http/`：客户端管理 API 控制器与模型。
- `client/configmgmt/`：客户端配置管理类型。
- `client/health/`：客户端健康检查逻辑。
- `client/event/`：客户端事件定义。
- `assets/`：Go 侧静态资源嵌入入口。

## 公共包

- `pkg/auth/`：认证实现，包括 token、OIDC、legacy 认证。
- `pkg/config/`：配置加载、模板、flag、v1 配置模型和 legacy 配置转换。
- `pkg/config/source/`：配置源、聚合、存储和校验。
- `pkg/errors/`：公共错误定义。
- `pkg/metrics/`：指标接口、内存指标、Prometheus 和聚合实现。
- `pkg/msg/`：控制消息定义、编码处理和 wire 测试。
- `pkg/naming/`：命名规则与名称生成。
- `pkg/nathole/`：NAT 穿透分析、发现、分类和控制器。
- `pkg/plugin/`：客户端插件、服务端插件和访问者插件机制。
- `pkg/policy/`：安全策略与 feature gate。
- `pkg/proto/`：底层协议实现，包括 UDP 和 wire 编解码。
- `pkg/sdk/`：对外 SDK 客户端。
- `pkg/ssh/`：SSH server、gateway 和 terminal 支撑。
- `pkg/transport/`：传输层消息与 TLS 支撑。
- `pkg/util/`：通用工具集合，包含网络、HTTP、日志、限速、指标、vhost、等待重试、版本等。
- `pkg/virtual/`：虚拟客户端相关能力。
- `pkg/vnet/`：虚拟网络、TUN 和消息控制逻辑。

## 配置与示例

- `conf/frps.toml`：服务端最小示例配置。
- `conf/frps_full_example.toml`：服务端完整示例配置。
- `conf/frpc.toml`：客户端最小示例配置。
- `conf/frpc_full_example.toml`：客户端完整示例配置。
- `conf/legacy/`：旧版 INI 配置示例，用于兼容性参考和迁移。

## Web 管理面板

- `web/package.json`、`web/package-lock.json`：Web workspace 根依赖与锁文件。
- `web/shared/`：`frpc` 与 `frps` 面板共享的 Vue 组件和 SCSS 变量。
- `web/frps/`：服务端管理面板，基于 Vue、TypeScript、Vite、Element Plus。
- `web/frps/src/api/`：服务端面板 API 调用。
- `web/frps/src/views/`：服务端面板页面。
- `web/frps/src/components/`：服务端面板组件。
- `web/frps/src/router/`：服务端面板路由。
- `web/frps/embed.go`、`web/frps/embed_stub.go`：服务端面板构建产物嵌入入口和 stub。
- `web/frpc/`：客户端管理面板，基于 Vue、TypeScript、Vite、Element Plus、Pinia。
- `web/frpc/src/api/`：客户端面板 API 调用。
- `web/frpc/src/views/`：客户端面板页面。
- `web/frpc/src/components/`：客户端面板组件，包含代理和访问者表单分段。
- `web/frpc/src/stores/`：客户端面板 Pinia 状态。
- `web/frpc/src/types/`：客户端面板类型定义和表单转换。
- `web/frpc/embed.go`、`web/frpc/embed_stub.go`：客户端面板构建产物嵌入入口和 stub。
- `web/**/node_modules/`：本地依赖目录，通常不手动修改。
- `web/**/dist/`：Web 构建产物目录，存在时会被 Go 嵌入。

## 测试

- `*_test.go`：各 Go 包的单元测试，通常与被测文件同目录。
- `test/e2e/`：端到端测试总目录，使用 Ginkgo/Gomega。
- `test/e2e/e2e_test.go`、`test/e2e/e2e.go`、`test/e2e/suites.go`：e2e 测试套件入口与组织。
- `test/e2e/framework/`：e2e 测试框架、进程管理、断言、请求和 mock server 调度。
- `test/e2e/mock/`：测试用 mock server。
- `test/e2e/pkg/`：e2e 测试辅助包，包括端口、请求、证书、进程、RPC、SSH、插件等。
- `test/e2e/v1/`：当前 v1 行为端到端用例。
- `test/e2e/legacy/`：旧版兼容行为端到端用例。
- `test/e2e/compatibility/`：跨版本兼容性测试。

## 定位建议

- 改 CLI 行为：先看 `cmd/frpc/`、`cmd/frpc/sub/` 或 `cmd/frps/`，再追到 `client/`、`server/`、`pkg/config/`。
- 改配置解析或校验：优先看 `pkg/config/`、`pkg/config/v1/`、`pkg/config/source/`。
- 改代理协议行为：同时检查 `client/proxy/`、`server/proxy/`、`pkg/msg/`、`pkg/proto/` 和 e2e 用例。
- 改管理 API：服务端看 `server/http/`，客户端看 `client/http/`，前端调用看 `web/frps/src/api/` 或 `web/frpc/src/api/`。
- 改 Web UI：先确认目标是 `web/frps/` 还是 `web/frpc/`，共享组件才进入 `web/shared/`。
- 改测试框架或 mock：看 `test/e2e/framework/`、`test/e2e/mock/`、`test/e2e/pkg/`。
